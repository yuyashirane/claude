### 🔴 実装フェーズ着手前の重要方針 3 点（独自解釈禁止）

詳細は指示書 §5.2 A-4 / A-5 を参照のこと。GO テキストでも要点を再掲：

#### 方針 1：`tax_code=None` は常に False 扱い

- `is_transitional_tax(None)` → False
- `is_full_deduction_tax(None)` → False
- 意味：判定不能は通常課税仕入にも経過措置にも入れない
- **`None in set` の挙動でブレない、防御ロジックを独自追加しない**

#### 方針 2：`Classification.NONE` の意味を 1 行で固定

```
NONE = 「課税仕入でもなく、インボイス論点の対象外」
```

- 非課税・対象外・判定不要を全部 NONE に集約
- `EXPECTED_*` と NONE は別概念：
  - `EXPECTED_*` = 分類は確定、Finding 化しないだけ（観察用）
  - `NONE` = そもそも分類対象外、観察も不要

#### 方針 3：`classify_transaction` は「必ず 1 つ返す」分類関数

```
1 取引 = 必ず 1 Classification（NONE 含む）
```

- フィルタ関数ではなく分類関数
- None や空タプルは返さない、必ず 6 値（5 分類 + NONE）のいずれか
- `sum(classification_counts.values()) == total_rows` が常に成立すること
- 不一致なら即報告レベル A
