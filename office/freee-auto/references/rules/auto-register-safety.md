# 自動登録の除外条件（CSVに含めない）

## 方針: 大胆に登録 → 事後チェックで修正

除外条件は最小限に絞り、80%の明細を自動処理する。
科目・税区分の精度は事後チェック（VERIFYステージ）で担保する。

## 除外条件（2条件のみ）

### 1. 複合仕訳が必要な取引

以下のキーワードを含む明細は自動登録しない:

| 種別 | キーワード |
|------|----------|
| 借入返済系 | 公庫、融資、ローン、借入、返済 |
| 給与系 | 給与、給料、賞与、ボーナス |
| 社会保険系 | 年金、健保、源泉税、厚生年金、健康保険、雇用保険、労災保険 |
| 口座振替系 | 相殺 |
| ATM・現金出金系 | ATM、現金出金 |

また、classificationステージで `excluded=true` が付与された明細（口座間振替・金額0等）も対象。

**理由**: これらは複数行の仕訳（元本+利息、支給額+控除等）が必要で、
単一行の自動登録ルールでは正しく処理できない。

### 2. 10万円以上の取引

`Math.abs(amount) >= 100,000` の取引は自動登録しない。

**理由**: 固定資産（資本的支出 vs 修繕費）の判定が必要。
10万円未満は全額経費処理で問題ないが、10万円以上は
固定資産台帳への登録要否を人が判断する必要がある。

## 旧除外条件（廃止済み）

以下の条件は事後チェック（VERIFYステージ）でカバーするため、
自動登録の除外条件からは削除した:

- ~~初出取引先~~ → 事後チェックで新規取引先を一覧レビュー
- ~~インボイス判定~~ → 事後チェックで税区分をレビュー
- ~~役員貸借・仮払仮受・交際費~~ → 事後チェックでピックアップ
- ~~雑費~~ → 事後チェックで科目レビュー
- ~~税務フラグ（🔴🟡）~~ → 事後チェックで税区分レビュー
- ~~既存ルールと矛盾~~ → 事後チェックで検出
- ~~高額取引（10万円以上30万円未満）~~ → 10万円で統一（以上を除外）

## 実装ファイル

- `src/register/rule-csv-generator.js` — routeForCsv() 内で上記条件をチェック
  - `COMPLEX_JOURNAL_KEYWORDS` 定数: 複合仕訳キーワード一覧
  - `isComplexJournalKeyword(description)`: キーワードチェック関数
  - `classification.excluded`: classificationステージの除外フラグも参照
