## §3. 観察セッションの目的（再確認）

### 3.1 最重要文

> **β2-C で実装した構造（Finding / FindingGroup / observations）が、複数社で「説明可能な世界」を提示しているかを観察する。**

### 3.2 核心の方針

- 「正しいか」ではなく **「世界の見え方が一貫しているか」** を見る
- 「綺麗な分布」ではなく **「説明できる分布」** を求める
- 検出ロジックの検証ではない（β2-B で完結済み）

### 3.3 観察の心構え（最重要）

> **観察は「結論を出すため」ではなく、「説明可能性を確認するため」に行う。**

特にパターン B（壊れてないが解像度不足）が出たときに有効：
- 「結論を出さなければ」と焦ると、無理に A や C に振り分けたくなる
- B のまま留めることも、観察の正しい結果

### パターン A（進んでよい）
- 不変条件：3 社全社成立
- classification：各社で意味のある分布（少なくとも 2 分類以上が非 0）

- 不変条件：成立
- classification：偏りあり
- observations：説明が曖昧 or 過不足あり
- **核心**：「壊れてはいないが、解像度が足りない」

### パターン C（止める）
- 不変条件：崩れる
- 同分類内で意味が揺れる（message 破綻含む）
- groups を見ても「何を意味しているか分からない」
- **核心**：「構造が嘘をついている状態」

## §7. 戦略 Claude の役割と制約（最重要）

### 7.1 戦略 Claude がやること

#### 観察進行のファシリテーション
- Claude Code への 3 社データ取得 GO
- JSON 取得確認 → 観察ログ枠作成
- 各社の基本 3 項目読み取り
- 3 社横断の傾向整理

#### 読み取りの型（5 項目、§6 観察セッション設計書から）
1. この会社では、どの分類が多い／少ないか
2. その分布は、会社の既知の性質から説明できるか
3. observations は、その説明を補助しているか
4. groups のサンプルは、分類名と意味が一致しているか
5. 追加で深掘りすべき点があるか

### 7.2 戦略 Claude がやらないこと（最重要）

#### A/B/C の最終判定をしない

理由：
- 戦略 Claude が A/B/C を提示すると、悠皓さんの判断が引っ張られる
- A/B/C の確定は税理士視点が必要（悠皓さんの責務）
- 戦略 Claude は構造的な読み取りに徹する役割

#### 例外

§5 の「3 社横断の読み取り」では **「パターン A / B / C のどれに近いか」の仮説提示** は許容（最終判定ではなく、論点整理のため）。ただし「これが結論だ」とは言わない。

### 7.3 戦略 Claude のミス傾向（前セッションで観察された注意点）

#### 注意 1：会話内で「指示書修正」を出してはいけない（事例 6 の教訓）

ファイル化されたドキュメント（観察セッション設計書）の方針と、会話内の指示が矛盾するのは事故の元。**観察セッション設計書を正本として尊重**する。

#### 注意 2：先回りして判断材料を整理しすぎない（Phase A 観察原則の継承）

Phase A 観察原則：「観察で得た違和感を解釈・切り分け・修正しない」。β2-C 観察は Phase A とは別性質だが、**戦略 Claude が先回りして整理しすぎると悠皓さんの判断を引っ張る**点は同じ。**読み取りに徹する**。

#### 注意 3：「綺麗」と「説明できる」を混同しない

数値が綺麗（極端な偏りがない、分布が均等）= 良い、と判断しない。partner_unknown が 0 件でも「なぜ 0 件か説明できる」なら A、説明できないなら B。**説明可能性が判断基準**。

## §12. 観察セッションで起きる可能性のあるシナリオ

### シナリオ A：3 社すべて分布が説明可能 → パターン A 確定

最も望ましい結果。β2-D 着手へ進む。

### シナリオ B：12243357 で qualified_* が 0 件、説明可能か微妙

「全件 qii=False」が観察軸として説明できれば A、できなければ B。**戦略 Claude は判定せず、悠皓さんの税理士視点で確定する**。

### シナリオ C：10794380 で経過措置コード過剰ヒット

userMemories の β2 残論点（β2-D 持ち越し）でもある。**β2-C 観察セッションでは「観察まで」で留め、対応は β2-D / β2-E に送る**のが正しい。観察セッション中に対応策を考え始めない。

### シナリオ D：不変条件が崩れる（パターン C）

3525430 では成立済みだが、12243357 / 10794380 で不変条件が崩れる場合：
- 即レベル A 報告
- β2-C 微修正の指示書作成
- Claude Code に GO

ただしこのシナリオの可能性は低い（β2-B / β2-C で classify_transaction の挙動は既に検証済み）。

### シナリオ E：観察中に違和感が増えすぎて 90 分を超える

→ **休憩を挟む**。集中力が落ちると判断の質が下がる。
→ 1 セッションで決められない場合は、観察を 2 セッションに分割する判断もあり得る。
