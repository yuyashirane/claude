## 次フェーズで守ること（最重要原則）

### 原則 1：論点前に現物

実装指示書執筆・粒度判断の **前に**、必ず現物を view する。「引き継ぎ書 v1 に書いてある」「memory に書いてある」は view したことにならない。

### 原則 2：「移すだけ、整理しない」

本タスクは TC_template.xlsx の移動 + path 1 行更新が中核。**それ以外の整理欲を発動させない**。

### 原則 3：運用判断は推測しない、聞く

backup ファイル / 「全パターン出力例」/ 既存運用との関係 等、判断に迷う箇所は **悠皓さんに 1 つ質問**する。事例 19 の予防。

### 原則 4：規模が小さいまま完了することを尊重する

本タスクは ~10〜30 行 + 数ファイル更新で完了する。**「もっと書くべきことがあるはず」と思って実装指示書を詳細化しすぎないこと**。L1-B の spec v3（1072 行）は本タスクには **過剰**。~200〜400 行で十分。
