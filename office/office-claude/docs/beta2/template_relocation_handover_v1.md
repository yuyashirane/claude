## 1. 本タスクの本質

### 1.1 タスク名

```
テンプレート配置ルール整備プロジェクト
TC_template.xlsx の templates/ 移動 + template_engine.py 参照先整理
```

### 1.2 本質

```
このプロジェクトは「移す」フェーズである。

L1-A：作る
L1-B：繋ぐ（bridge）
本タスク：移す（relocate）

データ・コード・ドキュメントの再配置のみ。
新規ロジック・新規設計判断は最小限。
```

### 1.3 背景（L1-B 完結セッションで発生した経緯）

L1-B 完結セッションのグループ 3（TC_template + V1-3-10 実機確認）で、悠皓さんから **ディレクトリ構造ルール（プラグイン互換）への準拠依頼** が提示された：

```
templates/                # レポート等に使うテンプレート
```

これは将来 `.claude/skills/` から `skills/` への機械的移送を見据えた構造で、**TC_template.xlsx も `templates/` に集約すべき**。

ただし、L1-B 完了判定（V1-3-10 実機 44/75/37 確認）と同時に実施すると検証の純度が崩れるため、**L1-B 完了後の独立タスクとして分離**された。

### 1.4 達成したいゴール

```
1. data/reports/template/ → templates/ への移動
2. template_engine.py の DEFAULT_TEMPLATE_PATH を新参照先に更新
3. 関連テスト・ドキュメントの同期更新
4. 既存運用（reports/3525430_アントレッド株式会社/ 等）への影響評価
5. baseline テスト（V1-3-20 129 / V1-3-10 211）維持
```

### 1.5 規模感（推定）

- 実装規模：~10〜30 行（path 定数 1 行 + 必要に応じた関連調整）
- 影響範囲：3〜10 ファイル
- 工数：1 セッション（2〜4 時間）で完結見込み

L1-B（spec v3 = 1072 行）と比較して **桁違いに小規模**。「壊さない最小スコープでの再配置」が本質。

## 4. 本タスクの最大のリスク（最重要）

### 4.1 範囲拡大の誘惑

本タスクは「移すだけ」のシンプルなタスクに見えるが、以下の **誘惑** が必然的に発生する：

| 誘惑 | 対処 |
|---|---|
| 「ついでに backup ファイルを整理したい」 | **やらない**。本タスクのスコープ外 |
| 「ついでに add_named_styles.py の所在も整理したい」 | **やらない**。別タスク |
| 「ついでに reports/ 配下のテンプレートも統合したい」 | **やらない**。運用配置との関係を別途精査すべき |
| 「ついでに他のテンプレートファイルも探して整理したい」 | **やらない**。grep 結果 ≠ 変更対象 |
| 「ついでに template_engine.py の他の path もきれいにしたい」 | **やらない**。最小スコープ |

### 4.2 影響範囲の見積もり困難

「templates/ に移すだけ」と言っても、以下が連動する可能性：

- template_engine.py の DEFAULT_TEMPLATE_PATH（必須改修）
- 関連テストの fixture / 期待値（要調査）
- ドキュメント（SKILL.md / RUNBOOK 等、要調査）
- **reports/.../TC_template.xlsx との関係**（要判断：これも触るか、触らないか）
- L1-B で配置済の `data/reports/template/` の他のファイル（backup / コピー / 全パターン出力例）の扱い（要判断）

### 4.3 既存運用への影響

`data/reports/3525430_アントレッド株式会社/` のような **会社別レポート出力ディレクトリ**との関係。これは：

- 出力先（生成 Excel の保存場所）
- 入力先（テンプレートの読み込み元）

両方の役割があり、**前者は本タスクのスコープ外、後者がスコープ内**。混同しないこと。

## 5. 本タスクで疑うべき前提

### 5.1 「templates/ にすべて集約」が正しいか？

これらは **着手前チェックで現物確認**してから設計判断する。

### 5.2 template_engine.py 以外で TC_template.xlsx を参照する箇所はあるか？

`DEFAULT_TEMPLATE_PATH` 以外に、以下の可能性：

- 環境変数経由でパス指定するコード
- 別の Skill から TC_template を参照するコード
- スクリプト類（`add_named_styles.py` 等）でハードコードされたパス

これらは grep で網羅的に確認する（ただし「grep ヒット ≠ 変更対象」原則を遵守）。

### 5.3 移動後の運用上の影響

- `data/reports/template/` を空にすべきか、残すか？
- `reports/3525430_アントレッド株式会社/TC_template.xlsx` との関係は？
- 業務 Excel レポート生成（V1-3-10 / V1-3-20）の挙動は変わらないか？

これらは **悠皓さんの運用判断** が必要な箇所。戦略 Claude が独断で決めない。
