# 提案書作成システム

## 概要
ヒアリング議事録から、税務顧問サービスの提案書（.docx）を自動生成するシステム。

## ワークフロー

### Step 1: 議事録の受領
ユーザーが議事録（テキスト、ファイル等）をアップロードする。

### Step 2: 議事録の分析 → config.json 生成
議事録から以下の情報を抽出・整理し、`config.json` を生成する：

1. **クライアント情報**: 会社名、業種、設立時期
2. **課題・ニーズ**: 議事録から読み取れる顧客の課題を整理
3. **サービス内容**: 顧客に適したサービスを選定
4. **料金プラン**: 売上規模・仕訳数に基づき `references/pricing/pricing-table.json` を参照して算出
5. **今後の流れ**: 顧客状況に合わせたステップ
6. **はじめに**: 議事録の内容を踏まえたカスタム挨拶文

### Step 3: 提案書の生成
```bash
node src/create-proposal.js <config.json> [output.docx]
```

### Step 4: 保存先
デフォルト: `G:\共有ドライブ\06_見込み客\{クライアント名}\税務顧問サービスのご提案_あしたの会計事務所_{クライアント名}.docx`

## 料金の算定ルール

`references/pricing/pricing-table.json` を参照して、以下の情報から料金を算出する：

| 項目 | 判定基準 |
|------|----------|
| 経理サポート月額 | 売上規模・仕訳数 |
| 決算申告報酬 | 売上規模 |
| 消費税等申告報酬 | 簡易課税 or 原則課税 |
| 年末調整 | 従業員数 |
| 法定調書 | 支払調書枚数 |
| 値引き | ご紹介、出精値引きの適用有無 |

### 議事録から読み取るべき情報
- 会社名、代表者名
- 業種・事業内容
- 設立時期（設立済み or これから設立）
- 売上規模（年商）、従業員数
- 月間仕訳数の見込み
- 現在の経理体制（自計化 or 記帳代行が必要）
- 消費税の課税区分（簡易課税 or 原則課税、免税事業者）
- 特別な課題やニーズ（消費税還付、海外取引、設立届出など）
- 紹介経由かどうか（紹介値引き適用の判定）

```
├── references/pricing/                 ← 料金表・顧客設定（config_*.json）

├── src/                                ← 実装コード
├── docs/                               ← 設計ドキュメント

└── logs/                               ← ログ
```
